# Jimmy Blog 个人化整理设计

## 目标

将 `jimmy6270/jimmy6270.github.io` 从原作者博客副本整理成 Jimmy 的个人博客。保留现有 Jekyll 主题及其必要的开源许可信息，删除原作者的文章、身份信息和第三方账号配置，并将站点定位调整为 AI 编程与应用。

## 公开身份

- 站名：`Jimmy Blog`
- 昵称：`Jimmy`
- 站点简介：`探索 AI 编程与应用，分享实践、工具与思考。`
- 正式地址：`https://blog.jebhenry.dpdns.org/`
- GitHub：`https://github.com/jimmy6270`
- 头像：`https://github.com/jimmy6270.png`

不公开邮箱、微博、知乎、Twitter 或其他未由 Jimmy 明确提供的账号。

## 内容处理

1. 删除 `_posts`、`_drafts` 等内容目录中的原作者文章。
2. 删除中文和英文 About 内容中的原作者经历、作品、演讲、媒体报道与社交链接。
3. 将 About 页面改为简短中文介绍，说明 Jimmy 关注 AI 编程、AI 工具和实际应用；不虚构教育、任职或项目经历。
4. 首页保留现有文章列表结构。没有新文章时允许显示空列表，不伪造示例文章。
5. 删除原作者友链。

## 配置处理

在 `_config.yml` 及相关模板中完成以下调整：

- 更新站名、SEO 标题、描述、关键词、正式域名和侧栏简介。
- 只保留 GitHub 社交入口。
- 使用 Jimmy 的 GitHub 头像。
- 删除或禁用原作者的 Disqus、Google Analytics、百度统计及其他第三方服务标识。
- 保持 `baseurl` 为空，以适配自定义域名根路径。
- 保留主题运行所需的 Jekyll、分页、Markdown、PWA 和样式配置，除非验证发现其依赖原作者服务。
- 确认仓库根目录的 `CNAME` 内容为 `blog.jebhenry.dpdns.org`。

## 原作者残留清理

对文本文件执行全局检查，重点清理：

- `Hux`、`Huxpro`、`黄玄`、`廖剑明`
- `huangxuan.me`、`huxpro@gmail.com`
- 原作者的社交账号、头像地址、统计 ID、评论短名称和友链
- 原作者专属的简介、文章署名及 SEO 关键词

主题许可证、许可证文件、主题来源说明和第三方开源版权声明不属于冒充信息，必须保留。README 可以改写为 Jimmy 博客的维护说明，但需要保留准确的主题来源和许可证说明。

## 文件与组件边界

- `_config.yml`：站点身份、域名、SEO、社交入口和第三方服务开关。
- `_includes/about/` 与 `about.html`：Jimmy 的公开介绍。
- `_posts/`、`_drafts/`：文章内容；本次清空原作者内容。
- `CNAME`：GitHub Pages 自定义域名。
- `README.md`：仓库用途、运行方式、主题来源和许可证。
- `_layouts/`、`_includes/`、静态资源：只修改仍直接展示原作者身份的部分，不做无关重构。

## 实施数据流

1. 将 GitHub 源仓库同步到当前本地工作区。
2. 盘点全部受版本控制的文本文件和内容目录。
3. 先清空原作者文章，再替换站点配置和 About 内容。
4. 清理模板、README 和静态资源引用中的身份残留。
5. 运行全局残留扫描。
6. 执行 Jekyll 构建或项目现有的等价构建检查。
7. 检查生成页面中的首页、About、GitHub 链接、头像和规范域名。

## 异常处理

- 如果本地无法安装或运行 Jekyll，仍执行配置语法、Liquid 引用、链接和残留文本检查，并明确报告未完成的构建验证。
- 如果仓库内存在无法确认归属的图片，只删除明显属于原作者个人身份或文章内容的图片；通用主题背景图继续保留。
- 如果删除文章后分页插件无法构建，则保留目录并调整空列表处理，不创建虚假文章。
- 如果自定义域名与现有 GitHub Pages 配置不一致，以用户确认的 `blog.jebhenry.dpdns.org` 和根目录 `CNAME` 为准。

## 验收标准

- 站点可识别的公开作者仅为 Jimmy。
- 首页、About、侧栏、页脚和元数据中不存在原作者身份或账号。
- 原作者文章全部删除。
- 仅展示 GitHub `jimmy6270` 社交入口和对应头像。
- 正式域名及 `CNAME` 均为 `blog.jebhenry.dpdns.org`。
- 原作者评论、统计和分析服务不会继续收集数据。
- 开源主题的必要许可与来源说明得到保留。
- 项目构建成功；若受本地环境限制，则所有可执行的静态检查通过并记录限制。
